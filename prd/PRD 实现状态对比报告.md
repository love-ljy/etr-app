## 四、缺失功能清单

### ❌ 高优先级缺失

1. **动态滑点自动提升机制**
   - 问题：`SlippageController.sol` 存在但未实现价格跌幅自动提升滑点
   - 影响：无法实现 PRD 要求的"价格跌幅≥10%/20%/30% 滑点自动提升"
   - 建议：需要补充 `SlippageController` 的价格监控和滑点调整逻辑

2. **滑点每日恢复 5%**
   - 问题：完全未实现
   - 影响：卖出滑点提升后无法自动恢复
   - 建议：在 `SlippageController` 中添加时间衰减机制

3. **小区业绩 10 层限制**
   - 问题：`DividendPool._calculateTeamPerformance()` 递归无层数限制
   - 影响：可能计算超出 PRD 要求的 10 层
   - 建议：添加 `maxDepth` 参数限制递归深度

4. **LP 权限丢弃机制**
   - 问题：LP 池持有 5% 代币，但未明确 LP 权限如何丢弃到黑洞
   - 影响：不符合 PRD"LP 权限丢弃打入黑洞"要求
   - 建议：需要明确 LP 代币的处理方式（是否需要添加到 PancakeSwap 等 DEX）

### ⚠️ 中优先级缺失

5. **分红历史查询**
   - 问题：`DividendPool` 合约无分红历史记录
   - 影响：用户无法查看历史分红明细
   - 建议：添加 `mapping(address => DividendRecord[])` 记录历史

6. **个人中心完善**
   - 问题：个人中心功能不完整
   - 影响：用户体验不佳
   - 建议：补充钱包管理、推荐码生成、设置等功能

7. **ETR 直接质押功能**
   - 问题：`CompoundPoolV2.depositETR()` 需要开关开启
   - 影响：用户无法直接质押 ETR（只能通过 USDT 购买）
   - 建议：明确产品策略，是否需要开启此功能

### 📝 低优先级优化

8. **价格预言机集成**
   - 问题：`PriceOracle.sol` 存在但未完全集成
   - 影响：ETR 价格依赖管理员手动更新
   - 建议：集成 Chainlink 或 PancakeSwap 价格喂价

9. **Subgraph 索引**
   - 问题：`subgraph/` 目录存在但未部署
   - 影响：前端查询依赖合约直接调用，效率低
   - 建议：部署 Subgraph 优化数据查询

## 五、链上数据 vs 后端需求

### 合约可记录的数据

| 数据类型 | 合约记录 | 查询方式 |
|---------|---------|---------|
| 用户质押记录 | ✅ `stakes[stakeId]` | 直接合约调用 |
| 用户收益明细 | ✅ `totalClaimed`, `lastClaimTime` | 直接合约调用 |
| 推荐关系 | ✅ `referrers[user]` | 直接合约调用 |
| 推荐统计 | ✅ `genCounts[referrer][generation]` | 直接合约调用 |
| 分红等级 | ✅ `userInfos[user].level` | 直接合约调用 |
| 历史交易 | ✅ 事件日志 | TheGraph/直接日志查询 |

### 需要后端/索引的数据

| 数据类型 | 原因 | 建议方案 |
|---------|------|---------|
| **用户资产变化趋势图** | 合约只记录当前状态，无历史快照 | 后端定时快照 or Subgraph |
| **团队业绩详细列表** | 链上查询 Gas 过高 | Subgraph 索引 |
| **分红历史记录** | 合约无存储历史 | 后端记录 or Subgraph |
| **推荐关系可视化** | 需要递归查询整条链 | Subgraph 优化查询 |
| **小区业绩计算** | 递归计算复杂且 Gas 高 | Subgraph 预计算 |
| **用户行为分析** | 合约无埋点 | 后端 Analytics |

### 推荐架构方案

```
┌─────────────────────────────────────────────────────┐
│                    前端 DApp                         │
└─────────────────────────────────────────────────────┘
         │                        │
         │ (实时交易)              │ (历史数据/复杂查询)
         ▼                        ▼
┌─────────────────┐      ┌─────────────────┐
│   合约直接调用   │      │   Subgraph      │
│   - 质押/解押    │      │   - 团队业绩     │
│   - 领取收益    │      │   - 分红历史     │
│   - 推荐绑定    │      │   - 趋势图表     │
└─────────────────┘      └─────────────────┘
                                │
                                ▼
                       ┌─────────────────┐
                       │  TheGraph Network│
                       │  (索引服务)      │
                       └─────────────────┘
```

**建议**: 
1. **优先使用 Subgraph** - 适合 DApp 的复杂查询需求
2. **后端仅做补充** - 用于 Subgraph 无法满足的定制需求
3. **事件驱动** - 监听合约事件更新索引
